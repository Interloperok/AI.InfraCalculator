# Models Structure

Эта директория содержит все Pydantic модели для AI Server Calculator API, организованные по функциональности.

## 📁 Структура

```
models/
├── __init__.py          # Импорты всех моделей
├── sizing.py           # Модели для расчета серверов
├── gpu.py              # Модели для GPU каталога
└── README.md           # Этот файл
```

## 📋 Модели по категориям

### Sizing Models (`sizing.py`)
Модели для расчета требований к серверам:

- **`SizingInput`** - входные параметры для расчета
- **`SizingOutput`** - результат расчета серверов
- **`WhatIfScenario`** - сценарий для анализа "что если"
- **`WhatIfRequest`** - запрос для анализа сценариев
- **`WhatIfResponseItem`** - элемент ответа для сценариев

### GPU Models (`gpu.py`)
Модели для GPU каталога:

- **`GPUInfo`** - информация о GPU
- **`GPUFilter`** - фильтры для поиска GPU
- **`GPUListResponse`** - ответ со списком GPU
- **`GPUStats`** - статистика по каталогу
- **`GPURefreshResponse`** - ответ на обновление данных

## 🔧 Использование

### Импорт всех моделей
```python
from models import SizingInput, SizingOutput, GPUInfo, GPUListResponse
```

### Импорт конкретной категории
```python
from models.sizing import SizingInput, SizingOutput
from models.gpu import GPUInfo, GPUListResponse
```

## ✨ Особенности

### Валидация данных
Все модели используют Pydantic валидаторы:
- Проверка диапазонов значений
- Валидация типов данных
- Кастомные валидаторы для бизнес-логики

### Документация Swagger
Каждая модель содержит:
- Подробные описания полей
- Примеры использования
- Схемы для Swagger UI

### Примеры в схемах
Все модели включают примеры для автоматической генерации документации.

## 🚀 Расширение

Для добавления новых моделей:

1. Создайте новый файл в директории `models/`
2. Определите Pydantic модели
3. Добавьте импорты в `__init__.py`
4. Обновите `__all__` список

## 📚 Связанные файлы

- `main.py` - основной API сервер
- `gpu_scraper.py` - скрапер GPU данных
- `contracts/` - старые модели (deprecated)
